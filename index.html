错误现象：
RuntimeError: The expanded size of the tensor (32) must match the existing size (1600) at non-singleton dimension 1.

排查思路：
模型加载成功，但在第一次 forward 时崩溃。通过堆栈定位到 prepare_action → pad_vector。发现输入向量维度 1600，目标维度 32，而原函数只处理了填充，没有截断逻辑。随手编写测试脚本直接调用 pad_vector(torch.randn(1,1600), 32) 复现错误。

修复补丁：

def pad_vector(vector, target_dim):
    current_dim = vector.shape[-1]
    if current_dim == target_dim:
        return vector
    elif current_dim < target_dim:
        pad_size = target_dim - current_dim
        pad = torch.zeros(*vector.shape[:-1], pad_size, device=vector.device, dtype=vector.dtype)
        return torch.cat([vector, pad], dim=-1)
    else:
        # 🔧 截断
        return vector[..., :target_dim]

错误现象：
RuntimeError: The size of tensor a (291) must match the size of tensor b (242) at non-singleton dimension 2

排查思路：
错误发生在注意力掩码计算时，两个掩码长度不同（291 vs 242）。分析源码发现函数假定两个掩码等长。添加打印形状后确认，并编写单元测试随机生成长度不同的布尔掩码，复现维度不匹配。

修复补丁：

def make_att_2d_masks(pad_masks, att_masks):
    pad_masks = pad_masks.bool()
    att_masks = att_masks.bool()
    if pad_masks.shape[-1] != att_masks.shape[-1]:
        min_len = min(pad_masks.shape[-1], att_masks.shape[-1])
        pad_masks = pad_masks[:, :min_len]
        att_masks = att_masks[:, :min_len]
    pad_2d_masks = pad_masks.unsqueeze(-2)
    att_2d_masks = att_masks.unsqueeze(-1)
    att_2d_masks = att_2d_masks & pad_2d_masks
    return att_2d_masks

错误现象：
AttributeError: TokenizersBackend has no attribute fake_image_token_id

排查思路：
模型初始化时在 smolvlm_with_expert.py 中访问 self.processor.tokenizer.fake_image_token_id 但属性不存在。检查 HuggingFace 官方 processor，确认默认不包含。采用动态补丁：在加载 processor 后立即检查并添加缺失属性。

修复补丁：

if not hasattr(self.processor.tokenizer, "fake_image_token_id"):
    self.processor.tokenizer.fake_image_token_id = 128002
if not hasattr(self.processor.tokenizer, "global_image_token_id"):
    self.processor.tokenizer.global_image_token_id = 128002

优化过程：
使用 PyTorch 内置 DistributedDataParallel 自动处理多卡通信。通过 nvidia-smi topo -m 确认四卡 NVLink 全互联（NV18）。尝试调整 NCCL 环境变量（NCCL_MAX_NCHANNELS, NCCL_ALGO 等），发现默认配置已经最优，加速比接近线性 3.84×。

分析结论：NVLink 带宽充足，通信开销极小，无需额外调优。

设计思路：
利用队列 + 双线程，推理线程持续计算结果放入队列，执行线程从队列获取最新动作立即执行。模拟机器人执行耗时 40ms/步，50 步任务端到端耗时从 4477ms 降至 2088ms。

核心代码片段：

def inference_worker():
    for _ in range(N):
        action = model(obs)
        queue.put(action)

def execution_worker():
    for _ in range(N):
        action = queue.get()
        robot.execute(action)   # time.sleep(0.04)
